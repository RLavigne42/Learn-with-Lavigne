## Capstone Objective
Demonstrate end-to-end professional workflow across setup, delivery, collaboration, complexity, and recovery.

## Project Scope
Create a small toy project (CLI utility or static web page) with:
- At least 2 issues
- At least 2 feature branches
- 1 conflict simulation + resolution evidence
- 1 recovery drill evidence
- 1 merged PR with clear narrative and validation

## Execution Checklist
- [ ] Issues created and linked in commits/PRs
- [ ] Branches follow naming convention
- [ ] Commits are meaningful and scoped
- [ ] PR template fully completed
- [ ] Review cycle simulated (request changes + update)
- [ ] Conflict resolved and documented
- [ ] Recovery drill completed and logged

## Acceptance Criteria
- Working feature outcome
- Clean history and maintainable PR
- Evidence bundle complete
