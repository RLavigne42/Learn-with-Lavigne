Use this checklist when reviewing or updating a PR.

## Change Quality
- [ ] Naming and intent are clear
- [ ] Scope is focused and cohesive
- [ ] No unrelated formatting churn

## Functional Confidence
- [ ] Tests updated or justification provided
- [ ] Validation commands listed and pass
- [ ] Edge cases acknowledged

## Docs & Ops
- [ ] README/docs updated when needed
- [ ] Migration/deploy notes provided if relevant
- [ ] Rollback path is realistic

## Security & Hygiene
- [ ] No secrets in code/config/history
- [ ] Access controls/token handling are compliant
- [ ] Sensitive logs omitted from PR

## Collaboration
- [ ] Reviewer comments addressed explicitly
- [ ] Follow-up actions tracked
- [ ] Re-review requested after updates
